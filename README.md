# TrueCheck — проверка текста на соответствие источникам

Веб-платформа, которая принимает текст (файл, вставку или **ссылку на статью**) и
его список литературы, разбивает текст на **эпизоды** (фрагменты со ссылками),
привязывает к каждому источник (загруженный файл → иначе догрузка из сети) и
проверяет, **действительно ли источник подтверждает написанное**. На выходе —
отчёт по всем эпизодам и сводный вердикт, включая оценку
«текст похож на ИИ-генерацию, а литература не соответствует тексту».

> Сделано по регламенту `/home/deploy/SkillTask.md` (каскад `spec → plan → build →
> test → review → ship`). Методология заимствована у проекта `fly`; **дизайн, код
> и логика — оригинальные.** Автор © 2026 @Zerno.

## Как это работает
1. **Разбор** (`extract.py`) — текст из txt/md/csv/html/rtf/odt/docx/doc/pdf/epub
   или по URL; отделение списка литературы; парсинг ссылок `[1]`, `[2, с. 14]`,
   `(Иванов, 2021)`, URL/DOI; нарезка на эпизоды.
2. **Источники** (`sources.py`) — гибрид: загруженный файл, иначе скачивание по
   DOI/URL. Недоступный источник честно помечается, а не выдаётся за «проверено».
3. **Сверка** — детерминированные эвристики (`heuristics.py`: наличие источника,
   дословные цитаты, пересечение терминов, автор/год) + **строгий ИИ-судья**
   (`verifier.py`) под изолированным правилом `AI_RULES.md`.
4. **Вердикт** (`grader.py`) — итог эпизода = ХУДШИЙ из (эвристики, ИИ); сводный
   балл соответствия 0–100% и оценка риска «ИИ-генерация/несоответствие».

### Анти-галлюцинация (главное)
ИИ **не сочиняет факты**: судит только по переданному фрагменту источника, обязан
привести **дословную цитату**. Код не верит модели на слово — если цитаты нет в
источнике, вердикт принудительно понижается (`verifier._apply_safeguards`,
см. `AI_RULES.md` §6). Это покрыто блокирующим тестом `test_verifier.py`.

## Запуск

### Docker (рекомендуется)
```bash
cd /home/deploy/check
cp .env.example .env   # если .env ещё нет; впишите ключи ИИ
docker compose up -d --build
# UI: http://<сервер>:8080
```

### Локально (dev)
```bash
cd /home/deploy/check
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000   # http://localhost:8000
```

### Тесты (всегда в изоляции данных, без живого ИИ)
```bash
DATA_DIR=/tmp/check-iso AI_ENABLE=false python -m pytest -q
```

## Конфигурация (`.env`)
| Переменная | Назначение |
|---|---|
| `AI_ENABLE`, `VERIFY_AI` | включение ИИ-сверки |
| `AI_PROVIDER` | `openai` (совм. шлюз) или `gemini` |
| `OPENAI_*`, `GEMINI_*` | ключи/эндпоинты (перенесены из fly) |
| `VERIFY_TEMPERATURE` | 0 — детерминизм |
| `VERIFY_MIN_QUOTE_CHARS` | мин. длина дословной цитаты для «подтверждён» |
| `FETCH_SOURCES` | догружать источники из сети, если нет файла |
| `MAX_UPLOAD_MB`, `DATA_DIR` | лимит загрузки, каталог данных |

## API
- `GET /` — веб-интерфейс.
- `POST /check` — `doc_file` | `doc_text` | `doc_url` + `source_files[]` + `use_ai`.
- `GET /report/{id}` — HTML-отчёт; `GET /api/report/{id}.json` — JSON.
- `GET /health` — статус и доступность ИИ.
