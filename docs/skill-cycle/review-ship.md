# TrueCheck — Review + Ship (go/no-go)

Каскад по `/home/deploy/SkillTask.md`: spec → plan → build → test → **review → ship**.

## Ревью по 5 осям
- **Корректность.** 30 тестов зелёные; парсинг ссылок/эпизодов, эвристики, ИИ-судья,
  грейдинг, e2e покрыты. Живой прогон с ИИ ловит смысловую подмену («50°C» → НЕ ПОДТВЕРЖДЁН),
  верную ссылку подтверждает дословной цитатой.
- **Читаемость.** Типизированные dataclasses, чистые функции, докстринги; ruff чист.
- **Архитектура.** Чистое ядро без сети/ИИ; сеть и ИИ изолированы (`sources/llm/verifier`).
  ИИ не может «поднять» статус выше эвристик (итог = худший).
- **Безопасность.** Исправлено в ревью: (1) редактирование секретов в логах (`llm._scrub`);
  (2) SSRF-защита загрузки URL (`sources._is_public_url` блокирует loopback/private/link-local/
  cloud-metadata, только http/https). Загрузки ограничены по размеру; пути отчётов санируются;
  Jinja2 автоэкранирование (нет XSS); секреты только в `.env` (gitignored).
- **Производительность.** ИИ — один вызов на эпизод (приемлемо); `temperature=0`.
  FYI/Optional: кэшировать токены источника между эпизодами; батчить ИИ-вызовы для крупных документов.

## Известные ограничения (зафиксировано)
1. **Мульти-цитатные эпизоды.** Если эпизод ссылается на несколько источников, статус
   считается по лучшему по совпадению. Недоступность одной из ссылок может маскироваться.
   → кандидат: статус по худшей из цитат / по-цитатная детализация.
2. **Режим без ИИ.** Чистые эвристики не ловят числовые/смысловые подмены при высоком
   пересечении терминов (даёт «частично»). Семантику обеспечивает ИИ-судья.
3. **Догрузка из сети.** Пейвол/JS-рендер страниц → текст может не извлечься (`not_found`),
   это честно помечается, а не выдаётся за «проверено».
4. **Квоты ИИ.** При 429/недоступности — безопасный фолбэк на эвристики (`ai_unavailable`).
5. **Парсинг ГОСТ-ссылок.** Регэкспы покрывают частые стили; экзотические — риск пропуска.

## Откат
- Стоп/деплой: `docker compose down` / `docker compose up -d --build`.
- ИИ-«kill switch»: `VERIFY_AI=false` в `.env` → только эвристики.
- Деплой обратим (нет внешних миграций; данные в томе `truecheck_data`).
