Модуль 23. MiMo Code (Xiaomi): установка на Windows и борьба с ошибками

Чему вы научитесь: устанавливать и запускать MiMo Code на Windows тремя способами и чинить типичные ошибки.

Зачем это нужно

MiMo Code (слоган «Where Models and Agents Co-Evolve») — это терминальный AI-агент кодинга от Xiaomi MiMo, концептуально близкий к Claude Code: читает и пишет код, выполняет команды, управляет Git и ведёт постоянную память. Полезно знать его как альтернативу и уметь установить на типичной пользовательской машине — под Windows, где как раз больше всего граблей.

ЗаметкаЧто внутри MiMo Code

Режимы-агенты: build (по умолчанию), plan (read-only), compose (оркестрация), переключение по Tab. Поддержка MCP, субагентов, постоянной памяти, голосового ввода. Каналы моделей: MiMo Auto (бесплатно ограниченное время, нулевая конфигурация), Xiaomi MiMo Platform (OAuth/оплата), Import from Claude Code (миграция авторизации) и Custom Provider (любой OpenAI-совместимый API, в т.ч. OpenRouter). По умолчанию работает через облачные модели семейства mimo-v2.x.

Предусловия (Windows)

  • Node.js 18 или новее (рекомендуется LTS 20/22) + npm. Проверка: node -v, npm -v.
  • Git (для работы с репозиториями и ручной установки скиллов).
  • Современный терминал: Windows Terminal или PowerShell 7.

Способ 1. npm на нативном Windows (рекомендуемый)

Официальное имя пакета — @mimo-ai/cli:

node -v                       # должно быть >= 18
npm install -g @mimo-ai/cli
mimo
ВниманиеНе путайте пакеты

Официальный пакет — именно @mimo-ai/cli. Похожий по названию @xiaomi-mimo/cli в npm — неофициальный, не ставьте его.

Способ 2. WSL2 (Ubuntu) + curl one-liner (самый надёжный)

Однострочник curl ... | bash нативно в cmd/PowerShell не работает (это Unix-путь). Зато прекрасно работает в WSL2 — подсистеме Linux внутри Windows. Это же лечит баг сетевого клиента (см. ниже).

wsl --install -d Ubuntu        # один раз, затем перезагрузка

Внутри Ubuntu (WSL):

# Node.js 18+ через nvm (без ошибок прав)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
source ~/.bashrc
nvm install 22

# Установка MiMo Code
curl -fsSL https://mimo.xiaomi.com/install | bash
mimo

Способ 3. Git Bash

В Git Bash есть curl и bash, но официальный скрипт ориентирован на macOS/Linux и может поставить не тот бинарник. В Git Bash на Windows надёжнее ставить через npm: npm install -g @mimo-ai/cli.

Обновление/удаление: mimo upgrade, mimo uninstall.

Большинство трудных багов MiMo Code на Windows (в частности ConnectionRefused из issue #920) обходятся переходом в WSL2. Это общее правило для многих CLI-инструментов, рождённых в Linux-экосистеме: WSL2 даёт «настоящий Linux» внутри Windows и снимает целый класс проблем с путями, окончаниями строк и сетевыми клиентами.

Первый запуск и конфигурация

Первый запуск (mimo) открывает мастер настройки с выбором канала: MiMo Auto / Xiaomi Platform (OAuth) / Import from Claude Code / Custom Provider. Полезные команды:

mimo auth login      # настроить ключи/OAuth
mimo auth list       # показать провайдеров
mimo models          # список моделей
mimo run "..."       # неинтерактивный запуск
mimo --continue      # продолжить последнюю сессию

Slash-команды в TUI: /help, /models, /login, /voice, /dream (извлечь знания в память проекта), /distill (упаковать рабочий процесс в скилл). Конфиги хранятся в ~/.config/mimocode/ и .mimocode/ в проекте; креды — в ~/.local/share/mimocode/auth.json.

Подключение OpenRouter / кастомного провайдера

MiMo Code поддерживает любой OpenAI-совместимый эндпоинт. Через мастер mimo auth login → Custom Provider укажите base URL (https://openrouter.ai/api/v1), ключ и модель. Так вы получаете тот же дешёвый каскад (часть IV), но в среде MiMo.

MiMo-Skills

У MiMo есть свой механизм скиллов (репозиторий XiaomiMiMo/MiMo-Skills, MIT). Установка через npx: npx skills add XiaomiMiMo/MiMo-Skills (флаги --list, -g, --skill <имя>). Требуется переменная MIMO_API_KEY. Свои скиллы можно создавать прямо из рабочего процесса командой /distill.

Чеклист модуля 23

  • Установил MiMo Code одним из трёх способов (npm / WSL2 / Git Bash).
  • Знаю обход issue #10 (платформенный бинарник) и #920 (WSL2 / Custom Provider).
  • Понимаю частые ошибки Windows (PATH, EACCES, Execution Policy, CRLF).
  • Умею подключить Custom Provider (OpenRouter).

Упражнения

  1. Установите MiMo Code на Windows-машине (или в WSL2) и сделайте первый запуск на канале MiMo Auto.
  2. Подключите Custom Provider к OpenRouter с дешёвой моделью и сравните поведение с Claude Code.

Модуль 24. Агенты с GitHub: Hermes, OpenClaw и кодинг-агенты

Чему вы научитесь: находить, оценивать и безопасно запускать open-source агентов с GitHub, подключать их к своим LLM.

Зачем это нужно

На GitHub — десятки сильных агентных проектов. Уметь их разворачивать и подключать к дешёвым моделям (OpenRouter/Ollama) — практический навык. Но запуск чужого агента, который имеет доступ к вашим файлам и мессенджерам, требует осторожности — этому тоже научимся.

ЗаметкаО звёздах и проверке

Цифры популярности (звёзды) ниже приведены по состоянию на 2026 год и могут отличаться. Главное — не звёзды, а что инструмент делает, активность проекта и безопасность запуска. Всегда проверяйте репозиторий сами.

Hermes (Nous Research) — агент-каркас и модели

Под «Hermes» обычно понимают связку от Nous Research:

  • NousResearch/hermes-agent — самообучающийся AI-агент («the agent that grows with you») с автономным созданием навыков и постоянной памятью. Работает как CLI, как шлюз для мессенджеров (Telegram, Discord, Slack, WhatsApp, Signal) и поднимает OpenAI-совместимый API-сервер. Поддерживает 200+ моделей через OpenRouter/OpenAI и др.
  • Модели Hermes 3 / Hermes 4 (open-weight, 14B/70B/405B) — то, чем агент «думает»; заточены под tool-calling и ролевых агентов.

Установка hermes-agent (Linux/macOS/WSL2):

curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash
# затем перезапустить shell и:
hermes

Подключение моделей: hermes setup (интерактивно), hermes model (выбор провайдера/модели), /model provider:model (на лету). Провайдеры: OpenRouter, OpenAI, Hugging Face и любой OpenAI-совместимый endpoint (значит, и Ollama).

Вниманиеhermes-agent — широкая поверхность атаки

Агент с доступом к вашим мессенджерам и cron — это серьёзные полномочия. Устанавливайте только в изолированном окружении, без доступа к чувствительным данным, и прочитайте установочный скрипт перед запуском (curl ... -o install.sh, просмотреть, потом bash install.sh).

OpenClaw — персональный ассистент

openclaw/openclaw (автор Peter Steinberger) — персональный AI-ассистент «на любой ОС», который подключается к вашим мессенджерам и выполняет действия в вашей цифровой среде; навыки из marketplace. Подключение кастомных LLM — через документацию провайдеров (есть поддержка OpenRouter, OpenAI-совместимо).

ВниманиеOpenClaw — это НЕ кодинг-агент

OpenClaw — слой персональной автоматизации (почта, календарь, мессенджеры), а не инструмент для написания кода уровня Claude Code. Если вам нужно писать код — смотрите кодинг-агенты ниже. И отнеситесь к безопасности серьёзно: автор публично признавался, что «ships code I don't read» — запускайте такой агент только в изолированном окружении без доступа к личным данным.

Как оценивать репозиторий перед запуском

  • Активность: свежие коммиты, дата последнего релиза, закрываются ли issue.
  • Лицензия: MIT/Apache-2.0 — свободно; «fair-code»/SSPL/BSL — ограничения на коммерческий self-host.
  • Поддержка провайдеров: ищите в README упоминания OpenAI-compatible base URL, OpenRouter, Ollama.
  • Способ установки: официальный Docker-образ — хороший знак (легче изолировать).
  • Красные флаги: установка только через curl | bash без исходников; запрос лишних прав; зашитые телеметрия/ключи.

Универсальный приём: подключить к OpenRouter или Ollama

Большинство агентов понимают OpenAI-совместимый интерфейс. Тогда достаточно переопределить базовый URL и ключ через переменные окружения:

# OpenRouter (300+ моделей, OpenAI-совместимо)
export OPENAI_API_KEY="sk-or-v1-..."
export OPENAI_BASE_URL="https://openrouter.ai/api/v1"
# в части инструментов переменная называется OPENAI_API_BASE — сверяйтесь с доками агента
# Ollama (локально)
export OPENAI_BASE_URL="http://localhost:11434/v1"
export OPENAI_API_KEY="ollama"

Сквозной мини-туториал: Aider + OpenRouter за 5 минут

Доведём один кодинг-агент до результата. Aider — простой терминальный pair-programming агент на Python.

  1. Установка:
python -m pip install aider-chat
  1. Ключ OpenRouter (в окружение, не в код):
export OPENROUTER_API_KEY="sk-or-v1-..."
  1. Запуск в каталоге проекта с дешёвой моделью (Aider понимает формат openrouter/<модель>):
cd my-project
aider --model openrouter/deepseek/deepseek-v3.2
  1. В открывшемся чате дайте задачу обычными словами:
> Добавь в app.py эндпоинт /health, возвращающий {"status": "ok"}, и тест на него.

Ожидаемый результат: Aider предложит точечные правки (diff), применит их к файлам, при желании закоммитит. Вы только что запустили стороннего агента с GitHub на дешёвой модели — тот же приём (openrouter/<модель> + ключ) работает и с opencode, Cline, Goose.

СоветТот же принцип для любого агента

Запомните схему: установить → задать OPENROUTER_API_KEY/OPENAI_BASE_URL → указать модель в формате провайдера → дать задачу. С небольшими отличиями в именах переменных она повторяется почти во всех агентах с GitHub.

Безопасный запуск стороннего агента (least-privilege)

  • Sandbox/Docker: запускайте агента в контейнере без доступа к хост-секретам; монтируйте только нужную папку: -v $(pwd)/project:/work.
  • Сеть: ограничьте исходящие соединения, если агент не должен ходить в интернет.
  • Узкие токены: выдавайте fine-grained GitHub-токены только на нужный репозиторий, а не личный токен с полным доступом.
  • Human-in-the-loop: держите подтверждение для rm, git push, деплоев. Не отключайте флаги подтверждения в незнакомом репозитории.
  • Читайте curl | bash-скрипты перед запуском.
  • Self-host на VPS: файрвол (ufw), доступ по SSH-ключам, обратный прокси с HTTPS, никаких портов агента наружу без аутентификации.
ВажноГлавное правило экосистемы

Любой агент с GitHub — это код с правами вашего пользователя. Относитесь к его запуску как к запуску чужой программы: изолируйте (Docker), ограничивайте права и сеть, не давайте секретов сверх необходимого. Удобство «одной командой» не должно отключать вашу осторожность.
