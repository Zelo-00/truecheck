Модуль 14. Карта провайдеров и моделей

Чему вы научитесь: ориентироваться в ландшафте LLM и понимать различие закрытых и открытых моделей.

Зачем это нужно

Моделей сотни, и они обновляются ежемесячно. Запоминать конкретные имена бессмысленно — устареют. Важно понимать категории и принципы выбора, которые останутся с вами.

Два мира: закрытые и открытые модели

  • Закрытые (proprietary) — Claude (Anthropic), GPT (OpenAI), Gemini (Google). Доступны только по API. Обычно лидируют по качеству на сложных задачах, но дороже, и вы не контролируете «железо».

Модуль 15. OpenRouter: один ключ на 300+ моделей

Чему вы научитесь: подключать OpenRouter как единый шлюз, управлять маршрутизацией, находить дешёвые и бесплатные модели и писать под них код.

Зачем это нужно

Заводить отдельный аккаунт и платёжку у каждого провайдера — мучительно. OpenRouter — это унифицированный, OpenAI-совместимый шлюз, который маршрутизирует запросы к 300+ моделям множества провайдеров через один API-ключ и один баланс. Сменить модель — это поменять одну строку в коде. Для каскада агентов, где разные роли требуют разных моделей, это идеально.

Как устроен биллинг OpenRouter

  • Кредитная система в долларах. Покупаете кредиты, списание по токенам по цене провайдера. На сам инференс OpenRouter наценки не делает.
  • Комиссия на пополнение ~5%. При оплате картой берётся около 5.5% (минимум $0.80) от суммы пополнения, криптой — 5%. Это разовая комиссия «на входе», а не на каждый запрос.
  • BYOK (Bring Your Own Key). Можно подключить собственные ключи провайдеров (первый 1 млн запросов в месяц бесплатно, далее 5% от стоимости) — если у вас уже есть прямой контракт, но нужен единый интерфейс.

Подключение: drop-in для OpenAI SDK

OpenRouter совместим с OpenAI API, поэтому любой инструмент, умеющий работать с OpenAI SDK, переключается сменой base_url и ключа.

  • Базовый URL: https://openrouter.ai/api/v1
  • Ключ: заводится в дашборде (openrouter.ai/keys).
# pip install openai
from openai import OpenAI

client = OpenAI(
    base_url="https://openrouter.ai/api/v1",
    api_key="OPENROUTER_API_KEY",   # держите в .env, не в коде!
)

resp = client.chat.completions.create(
    model="deepseek/deepseek-v3.2",     # любой id с openrouter.ai/models
    messages=[{"role": "user", "content": "Что такое каскад агентов в двух словах?"}],
    extra_headers={
        "HTTP-Referer": "https://my-course.example",  # опц., для рейтингов
        "X-Title": "AI Agents Course",                # опц., имя приложения
    },
)
print(resp.choices[0].message.content)

Маршрутизация (routing) — тонкая настройка

OpenRouter позволяет управлять тем, какой провайдер обслужит запрос, через объект provider или суффиксы в имени модели:

ПараметрЧто делает
sort: "price" / "throughput" / "latency"Сортировать по цене / скорости / задержке
order: [...]Список провайдеров по порядку предпочтения
only / ignoreБелый / чёрный список провайдеров
max_priceПотолок цены: {"prompt": 1, "completion": 2} ($/М)
data_collection: "deny"Запретить провайдеров, логирующих данные
allow_fallbacks: trueАвтоматически переключаться на запасного при сбое

Суффиксы-шорткаты: :floor (минимальная цена), :nitro (максимальная скорость), :free (бесплатный вариант).

resp = client.chat.completions.create(
    model="qwen/qwen3.6-35b-a3b",
    messages=[...],
    extra_body={"provider": {
        "sort": "price",
        "allow_fallbacks": True,
        "data_collection": "deny",
        "max_price": {"prompt": 0.5, "completion": 1.5},  # $/М
    }},
)
СоветАвто-fallback спасает каскад

Для агентов в проде задавайте order с запасным провайдером и allow_fallbacks: true. Тогда сбой одного провайдера не уронит весь ваш каскад — запрос автоматически уйдёт к следующему.

Бесплатные модели (тариф :free)

У многих моделей есть вариант с суффиксом :free — инференс бесплатен, но с жёсткими лимитами:

  • ~20 запросов в минуту;
  • 50 запросов в сутки, если на аккаунте куплено меньше $10 кредитов;
  • 1000 запросов в сутки, если куплено ≥ $10 кредитов (то есть небольшое пополнение резко расширяет «бесплатный» режим);
  • неудачные запросы тоже списывают дневную квоту.

Среди бесплатных на июнь 2026 встречаются варианты Qwen, Llama 3.3 70B, Gemma, GPT-OSS, NVIDIA Nemotron, Hermes 3 405B и др. Состав постоянно меняется — смотрите на openrouter.ai/models с фильтром по нулевой цене.

ЗаметкаГде уместен :free

Бесплатный уровень отлично подходит для прототипов, обучения и нерегулярных задач. Для агентов в проде он ненадёжен (лимиты, плавающая доступность) — там берите дешёвые платные модели, оставив :free для офлайн-экспериментов.

Выбор модели под роль каскада (через призму KSCR)

Главный принцип (вспомните каскад и KSCR): дешёвая модель делает 90% объёма, дорогая — только трудные шаги. Выбор под роль — это управление коэффициентом трансляции η: не грузить слабую модель сложным промптом (иначе η падает), и не «стрелять из пушки по воробьям».

Стратегия удешевления (6 рычагов):

  1. Дешёвый черновик → дешёвая проверка → эскалация. Воркер генерирует кандидат, дешёвый судья проверяет, на сильную модель уходит только провал.
  2. Router-first. Лёгкий классификатор сортирует входящие: простое → самой дешёвой модели, сложное → планировщику.
  3. Жёсткий max_price и :floor на «расходных» ролях.
  4. Prompt caching — кэшируйте неизменную часть (системный промпт, инструменты): экономия 60–80% на повторяющемся контексте.
  5. Ограничивайте output-токены — выход дороже входа в 3–6 раз; просите краткость и структуру (JSON-схемы).
  6. Fallback-цепочки — чтобы агент не падал при сбое провайдера.

Готовые сниппеты: роутер по сложности и retry

Два кусочка кода, которые понадобятся в каскаде. Роутер выбирает дешёвую или сильную модель по эвристике (длина + ключевые слова):

CHEAP = "mistralai/mistral-nemo"          # классификация, простое
STRONG = "deepseek/deepseek-v3.2"         # рассуждение, код, длинное

def pick_model(task: str) -> str:
    hard = any(k in task.lower() for k in
               ("код", "code", "проанализируй", "спроектируй", "докажи", "почему"))
    return STRONG if (hard or len(task) > 600) else CHEAP

Retry-обёртка с экспоненциальной задержкой на 429/5xx (чтобы агент не падал при перегрузке провайдера):

import time, random

def call_with_retry(client, max_retries=5, base=1.0, **kwargs):
    for attempt in range(max_retries):
        try:
            return client.chat.completions.create(**kwargs)
        except Exception as e:
            code = getattr(e, "status_code", None)
            if code not in (429, 500, 502, 503) and code is not None:
                raise                      # клиентские ошибки не ретраим
            time.sleep(min(base * 2 ** attempt + random.random(), 30))
    raise RuntimeError("исчерпаны попытки")
СоветРоутер экономит больше всего

Лёгкий роутер «отсекает» простые запросы на самую дешёвую модель — и большинство задач вообще не доходит до дорогой. В паре с fallback-цепочками OpenRouter это и дёшево, и надёжно.

Модуль 16. Локальные модели и альтернативы OpenRouter

Чему вы научитесь: запускать модели локально через Ollama, знать бесплатные тарифы провайдеров и выбирать между облаком и своим железом.

Зачем это нужно

Иногда нужны приватность (данные не должны уходить наружу), офлайн или нулевая цена за токен. Тогда модель крутят локально. А ещё у многих провайдеров есть щедрые бесплатные тарифы для прототипов.

Ollama — модели на своём железе

Ollama — простейший способ запустить open-модели (Llama, Qwen, Mistral, DeepSeek-distill, Gemma, Phi) локально. Установка на Linux:

curl -fsSL https://ollama.com/install.sh | sh
ollama run qwen3:8b        # скачает и запустит модель

Главное для нас: Ollama даёт OpenAI-совместимый endpoint на http://localhost:11434/v1, поэтому весь код из модуля 15 работает сменой base_url:

client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")

Компромиссы: нужна железка (GPU или много RAM); крупные модели не влезут на слабый VPS; качество ниже фронтир-моделей; аптайм и масштабирование — на вас. На обычном VPS без GPU реалистично крутить только небольшие модели (3B–8B) и медленно. Для серьёзного локального инференса берут сервер с GPU.

ВниманиеЛокальные модели на VPS без GPU

Не ждите чудес от CPU-инференса: модель 7B на CPU отвечает десятки секунд. Для агентов, где важна скорость, дешёвая облачная модель (Mistral Nemo за $0.02/М) почти всегда практичнее локальной на слабом сервере. Локальный путь оправдан при требовании приватности/офлайна или при наличии GPU.

Когда что выбирать

  • OpenRouter — когда нужен один ключ на 300+ моделей, авто-fallback и быстрая смена модели по ролям каскада. Платите ~5% на пополнении, инференс — по цене провайдера.
  • Прямые free-tier (Google/Mistral/Groq/Cerebras) — для прототипов и нерегулярных задач, где лимиты приемлемы и не критична приватность.
  • Groq/Cerebras/Fireworks — когда критична скорость (интерактивные агенты).
  • Ollama/локально — когда важны приватность, офлайн, нулевая цена за токен и есть железо.
