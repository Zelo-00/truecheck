ЧАСТЬ VIII. КАПСТОУН И ПРИЛОЖЕНИЯ

Знания закрепляются только в проекте. Эта часть превращает курс в навык: вы соберёте собственный каскад агентов от начала до конца, а приложения дадут вам шпаргалки, шаблоны и словарь, к которым вы будете возвращаться в работе.

Модуль 25. Капстоун-проект: ваш автономный каскад

Чему вы научитесь: объединять все навыки курса в один работающий проект.

Задача капстоуна

Постройте автономный каскад агентов на вашем VPS, который решает реальную вашу задачу (примеры: сортировка и ответы на входящие письма; мониторинг и саммари новостей по теме; первичная поддержка пользователей; сбор и структурирование данных по расписанию). Требования:

  1. Развёрнут на защищённом VPS (часть I).
  2. Использует каскад из ≥3 ролей на преимущественно дешёвых моделях (части III, IV).
  3. Имеет фронт (вебхук, Telegram или расписание) и работает как сервис (часть VI).
  4. Каждый агент спроектирован по изолированным правилам, а его промпт доведён до C ≥ 0.8 по KSCR (часть III).
  5. Имеет мониторинг, лимиты расхода и бэкап (части I, VI).

Пошаговый план

Этап 1. Спецификация (день 1). Опишите задачу, входы, выходы, критерии успеха. Используйте скилл spec в Claude Code. Сформулируйте, какие роли нужны (роутер, воркеры, судья).

Этап 2. Инфраструктура (день 1–2). Поднимите/проверьте VPS: SSH-ключи, ufw, Docker, swap, .env с секретами и chmod 600. Разверните Postgres (память/очередь) и, если выбрали бескодовый путь, n8n.

Этап 3. Промпты по KSCR (день 2–3). Для каждой роли напишите изолированный свод правил (роль, полномочия, запреты, протокол неопределённости). Прогоните каждый промпт по KSCR-PDCA до C ≥ 0.8: проверьте полноту (8 пунктов CSP), устраните двусмысленность, не перегружайте (риск G>0 — тогда дробите роль).

Этап 4. Сборка каскада (день 3–5). Реализуйте оркестрацию: n8n (супервизор + под-воркфлоу) или свой код (FastAPI-шлюз + воркеры под systemd + очередь). Подключите модели через OpenRouter под каждую роль. Заложите fallback-цепочки.

Этап 5. Фронт и деплой (день 5–6). Подключите фронт (Telegram-бот / вебхук / cron). Оформите всё как сервисы (systemd/Docker с автоперезапуском). Закройте порты, поставьте HTTPS-прокси при необходимости.

Этап 6. Мониторинг и эксплуатация (день 6–7). Настройте логи, лимиты расхода на ключах, ежедневный взгляд на здоровье (df -h, docker ps), бэкап БД по cron. Прогоните 50–100 реальных событий, замерьте расход токенов и качество.

Критерии «сдано»

  • Система проработала сутки без вашего вмешательства.
  • Тяжёлая модель вызывалась < 10% случаев (каскад экономит).
  • Каждый агент имеет изолированные права (не больше нужного).
  • Промпты задокументированы с метриками KSCR.
  • Есть бэкап и лимиты расхода.

Приложение C. Глоссарий

  • Агент (AI-агент) — система LLM + инструменты + память + цикл, которая сама ведёт многошаговую работу до результата.
  • VPS — виртуальный частный сервер в аренду; постоянно включён, имеет белый IP.
  • SSH — протокол безопасного удалённого управления сервером.
  • Docker / контейнер / образ / том — упаковка приложения с зависимостями в изолированную среду; образ — слепок, контейнер — запущенный экземпляр, том — постоянное хранилище.
  • Claude Code — терминальный агент Anthropic для разработки.
  • CLAUDE.md — файл памяти проекта, который агент читает автоматически.
  • Скилл (Skill) — упакованная экспертиза (папка с SKILL.md), подгружается по необходимости (прогрессивное раскрытие).
  • Субагент — отдельный агент со своей ролью, инструментами, моделью и контекстом; вызывается оркестратором.
  • Хук (hook) — команда, выполняемая харнессом автоматически на событие.
  • Харнесс — программная обвязка (Claude Code, Agent SDK, n8n), которая реально исполняет вызовы инструментов модели и возвращает ей наблюдения; сама модель ничего не исполняет.
  • Токен — единица текста для LLM (примерно ¾ слова); по токенам считается и контекст, и стоимость.
  • temperature — параметр «креативности» генерации: выше — разнообразнее и менее предсказуемо, ниже — стабильнее.
  • Fallback-цепочка — список запасных провайдеров/моделей: при сбое одного запрос автоматически уходит к следующему.
  • MCP (Model Context Protocol) — стандарт подключения агента к внешним инструментам и данным.
  • ReAct — паттерн цикла агента: Reasoning (рассуждение) + Acting (действие) + Observation (наблюдение).
  • Оркестрация — координация нескольких агентов (конвейер, супервизор, каскад, дебаты).
  • Каскад — топология, где дешёвые модели делают объём, дорогая — только трудные шаги.
  • Изолированные правила — свод правил агента (роль, полномочия, запреты), с изоляцией контекста и прав.
  • KSCR — авторская методика оптимизации промптов: Knowledge → Skill → Competence → Rate. Метрики K, S, G, η, C.
  • CSP — чеклист семантической полноты промпта (8 пунктов) в KSCR.
  • ТКОП — трёхкомпонентный оптимизатор промптов (Оператор → Ассистент → Целевой промпт) — пример изоляции.
  • OpenRouter — OpenAI-совместимый шлюз к 300+ моделям через один ключ.
  • Промпт-кэширование — кэш повторяющегося контекста; кэш-чтение ~10% цены.
  • RAG — Retrieval-Augmented Generation: ответ модели на основе извлечённых из хранилища документов.
  • n8n — open-source платформа визуальной автоматизации с поддержкой ИИ-агентов.
  • queue mode — режим n8n с очередью на Redis и воркерами для масштабирования.
  • Agent SDK — библиотека-движок Claude Code для встраивания агента в свой код.
  • Managed Agents — серверная поверхность Anthropic: агент один раз, сессии на каждый запуск.
  • MiMo Code — терминальный AI-агент кодинга от Xiaomi.
  • Hermes / OpenClaw — агентные проекты с GitHub (каркас+модели Nous Research / персональный ассистент).

Приложение E. Быстрый старт за 1 день

Полный курс — это 4–6 недель. Но «быструю победу» можно получить за день: к вечеру у вас будет личный агент в Telegram на дешёвой модели. Это не замена полного трека — это мотивация и проверка, что всё складывается.

Утро — сервер и доступ (1–2 ч). Возьмите VPS (Часть I), подключитесь по SSH, обновите систему, создайте пользователя с sudo, настройте вход по ключу (модуль 2 — минимально). Поставьте Python и pip.

День — ключи и каркас (1–2 ч). Заведите ключ OpenRouter (часть IV). Положите в .env: OPENROUTER_API_KEY, TELEGRAM_BOT_TOKEN (токен у @BotFather), chmod 600 .env. Установите зависимости: pip install openai python-telegram-bot python-dotenv.

Вечер — агент в Telegram (1–2 ч). Соберите бота из модуля 21 (Telegram-фронт) + дешёвая модель через OpenRouter. Минимальный app.py:

import os
from openai import OpenAI
from telegram.ext import Application, MessageHandler, filters
from dotenv import load_dotenv
load_dotenv()

llm = OpenAI(base_url="https://openrouter.ai/api/v1",
             api_key=os.environ["OPENROUTER_API_KEY"])
ALLOWED = {123456789}   # ваш Telegram user id

async def on_message(update, context):
    if update.effective_user.id not in ALLOWED:
        return
    r = llm.chat.completions.create(
        model="deepseek/deepseek-v3.2",
        messages=[{"role": "user", "content": update.message.text}])
    await update.message.reply_text(r.choices[0].message.content)

app = Application.builder().token(os.environ["TELEGRAM_BOT_TOKEN"]).build()
app.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, on_message))
app.run_polling()

Запустите (python app.py), напишите боту — он ответит через дешёвую LLM. Готово: вы построили работающего агента за день. Дальше оформите его как systemd-сервис (модуль 21) и идите по полному треку, наращивая до каскада.

Частые вопросы (FAQ)

  • Подписка Claude или API-ключ? Для плотной личной работы — подписка (фиксированная цена); для серверной автоматизации и команд — API-ключ (прозрачно по токенам). Можно совмещать.
  • Почему мой :free упёрся в лимит? Бесплатный тариф OpenRouter: ~20/мин, 50/сутки при балансе < $10 (1000/сутки при ≥ $10). Неудачные запросы тоже списывают квоту.
  • Локальная модель или облако? Без GPU локальный инференс медленный — дешёвая облачная модель почти всегда практичнее. Локально — ради приватности/офлайна или при наличии GPU.
  • Как понять, что пора апгрейдить VPS? Если сервер постоянно сидит в swap (free -h), диск забивается логами (df -h), а контейнеры падают по OOM — пора.
  • Сколько стоит каскад? Считайте по формуле из модуля 22. Обычно бюджет «съедают» судья и эскалация — туда направляйте оптимизацию.
  • С чего начать, если страшно? С Приложения E «Быстрый старт за 1 день» — получите работающего агента сегодня, остальное освоите по дороге.

Заключение

Вы прошли путь от пустого VPS до автономного каскада ИИ-агентов. Соберём главное:

  1. Инфраструктура — это фундамент, а не «потом». Безопасный VPS (ключи, файрвол, Docker, секреты) экономит вам недели разбирательств.
  2. Claude Code — не чат, а рабочая станция: память проекта, скиллы, субагенты, хуки, MCP. Кто умеет инженерить контекст — у того ИИ пишет как senior.
  3. Теория делает вас инженером. Компоненты агента, цикл ReAct, топологии оркестрации, изолированные правила и измеримая методика KSCR — это оптика, через которую вы проектируете системы, а не копируете чужие.
  4. Экономика решает. Каскад на дешёвых моделях (OpenRouter, DeepSeek, Qwen, Mistral, Hermes) с дорогой моделью только на трудных шагах — это разница между «дорого и нестабильно» и «копейки и надёжно».
  5. Автоматизация без кода (n8n) и с кодом (Agent SDK) — два пути к одной цели. Выбирайте по задаче, комбинируйте.
  6. Экосистема растёт. MiMo Code, Hermes, OpenClaw и десятки агентов с GitHub — умейте их ставить, подключать к своим моделям и запускать безопасно.

Главный принцип, который останется с вами, когда устареют все конкретные модели и цены: проектируйте измеримо, изолируйте строго, экономьте каскадом, проверяйте всегда. Это и есть AI-инжиниринг.

Теперь — стройте. И помните KSCR: знание без умения — иллюзия полноты. Закрепляйте каждый модуль практикой на своём сервере.
