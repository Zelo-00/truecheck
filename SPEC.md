# Spec: «Сверка» — платформа проверки текста на соответствие источникам

> Разработка ведётся по общему регламенту `/home/deploy/SkillTask.md`
> (каскад `spec → plan → build → test → review → simplify → ship`).
> Методология взята из проекта `fly`; **дизайн, код и логика — оригинальные,
> не копия `fly`.**

## Допущения (поправьте — иначе работаю по ним)
1. Проверяемые тексты — преимущественно русскоязычные академические/новостные
   материалы со списком литературы и ссылками вида `[1]`, `[12, с. 44]`,
   `(Иванов, 2021)`, иногда сноски/URL/DOI.
2. «Эпизод» = смысловой фрагмент текста, к которому привязана ссылка на
   источник (предложение или абзац с маркером цитирования).
3. «100% соответствие» проверяется честно только когда текст источника доступен
   (загружен файлом или скачан). Если источник недоступен — это явно помечается,
   а не выдаётся за «проверено».
4. ИИ используется **только как судья соответствия** под строгим изолированным
   правилом (`AI_RULES.md`), а не как генератор контента.
5. Один пользователь/небольшая команда, без публичной мультиарендности на старте.

## Objective
Веб-платформа, которая принимает текст и его список источников, разбивает текст
на **эпизоды** (фрагменты с ссылками), по каждому эпизоду достаёт привязанный
источник (загруженный файл → иначе из сети) и проверяет, **действительно ли
источник подтверждает то, что написано в тексте**. На выходе — отчёт по всем
эпизодам и сводный вердикт, включая оценку «текст похож на ИИ-генерацию, а
литература не соответствует тексту».

Пользователи: преподаватели/рецензенты/редакторы/проверяющие, которым нужно
быстро увидеть, где ссылки «фейковые» (источник не существует, не содержит
утверждаемого, приписан не тому автору/году, выдуман целиком).

Success-критерии (тестируемые):
- Из текста корректно извлекаются ≥95% явных маркеров цитирования на эталонном наборе.
- Каждому эпизоду присвоен один из статусов: `ПОДТВЕРЖДЁН / ЧАСТИЧНО / НЕ ПОДТВЕРЖДЁН / ИСТОЧНИК НЕ НАЙДЕН / ИСТОЧНИК ВЫДУМАН`.
- Для статуса `ПОДТВЕРЖДЁН` в отчёте всегда есть **дословная цитата** из источника (≥ `VERIFY_MIN_QUOTE_CHARS` символов).
- ИИ ни при каких входных данных не возвращает факты/цитаты, которых нет в переданном фрагменте источника (проверяется тестом на «приманку»).
- Отчёт покрывает **все** эпизоды (ни один не пропущен) и даёт сводный балл соответствия 0–100%.
- Платформа поднимается одной командой `docker compose up -d --build`.

## Tech Stack
- Python 3.11, **FastAPI** + Uvicorn (API + серверный рендер через Jinja2).
- Парсинг: `python-docx` (docx), `pypdf` (pdf), встроенный текст (txt/md).
- Сеть/догрузка источников: `httpx`/`requests` (+ опц. `trafilatura` для извлечения текста со страниц).
- ИИ: собственный провайдер-агностик клиент `llm.py` (OpenAI-совместимый шлюз + Gemini fallback), ключи из `.env` (перенесены из `fly`).
- Хранилище: файловая система `DATA_DIR` (загрузки, кэш источников, отчёты) + SQLite-индекс заданий.
- Фронт: **свой** дизайн (светлая «криминалистическо-редакторская» тема, сплит-вью «текст ↔ вердикты»), без переиспользования стилей `fly`.
- Инфраструктура: nginx (reverse-proxy/статика) + app (uvicorn) в `docker-compose.yml`.

## Code Style
Чистые dataclass-модели, чистые функции движка (легко тестировать), ИИ изолирован
за одним интерфейсом. Пример контракта ИИ-судьи:
```python
@dataclass(frozen=True)
class MatchJudgement:
    status: Literal["SUPPORTED", "PARTIAL", "NOT_SUPPORTED", "SOURCE_MISSING"]
    quote: str          # ДОСЛОВНАЯ подстрока источника; "" если нечего цитировать
    rationale: str      # 1–2 предложения, только на основе фрагмента
    confidence: float   # 0..1

def judge(claim: str, source_excerpt: str | None) -> MatchJudgement:
    """Строгий судья. Если source_excerpt пуст → SOURCE_MISSING (никаких догадок).
    quote обязан быть дословной подстрокой source_excerpt, иначе статус понижается."""
```
Конвенции: snake_case, типы обязательны в движке, никаких сетевых вызовов в чистых
функциях (сеть только в `sources.py`/`llm.py`), русские строки в UI/отчётах.

## Testing Strategy
- `pytest`, тесты в `app/tests/`, прогон **всегда** с изоляцией `DATA_DIR=/tmp/check-iso`.
- Уровни: юнит (extract/heuristics/grader), контрактные (verifier на mock-LLM, включая «приманку» на галлюцинацию), e2e (загрузка → отчёт) на FastAPI `TestClient`.
- Обязательные кейсы: выдуманный источник, источник-есть-но-не-про-то, верная ссылка, смещённый автор/год, цитата дословная vs перефраз, источник недоступен.
- Цель покрытия движка ≥80%; «приманочный» тест на анти-галлюцинацию обязателен и блокирует релиз.

## Boundaries
- **Always:** держать тесты зелёными перед след. фазой; изоляция данных; дословная цитата для `ПОДТВЕРЖДЁН`; секреты только в `.env`; уникальный дизайн/логика.
- **Ask first:** новые внешние зависимости; смена схемы SQLite; включение реальных сетевых вызовов в CI; изменение правила `AI_RULES.md`.
- **Never:** коммитить `.env`/ключи; позволять ИИ выдавать факты вне переданного фрагмента; выдавать «не найдено» за «подтверждено»; копировать дизайн/код `fly`; `git push` без просьбы.

## Success Criteria
См. раздел Objective (тестируемый список). Релиз = `/ship` go: контейнеры
